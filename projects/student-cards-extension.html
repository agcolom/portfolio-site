The Technical Problem

The University of Westminster uses SITS (Student Information Technology System) for managing student records. When the university updated their SITS implementation to use DataTables with server-side pagination, several challenges emerged:

  • Complex Pagination: Student lists were split across multiple pages (10 students per page), requiring manual navigation through dozens of pages for large classes
  • Two-Level Pagination: For classes with 100+ students, there was a top-level navigation (groups of 100) and bottom-level navigation (pages of 10), making data extraction extremely tedious
  • Multiple Data Sources: Staff needed to access student data from three different SITS views (module lists, seminar groups, and student search results), each with different HTML structures
  • Time-Consuming Manual Process: Creating student cards manually meant copying data, formatting images, and creating documents for potentially hundreds of students

The Solution

I developed a Chrome extension that automatically:

  • Detects the current SITS page type and adapts extraction accordingly
  • Automatically navigates through DataTables pagination to collect all student records (up to 100 per group)
  • Extracts student photos, names, IDs, courses, and enrolment status
  • Generates professional PDF cards with 20 cards per page in landscape format
  • Includes intelligent headers showing module information, group numbers, and page numbers
  • Handles groups larger than 100 students by providing clear instructions for manual navigation

Key Technical Challenges

Challenge: DataTables Pagination

SITS uses DataTables with AJAX pagination, showing only 10 students at a time. The extension needed to automatically click through all pages and detect when pagination was complete.

Solution: Smart Polling

Implemented a polling mechanism that tracks the first student ID on each page, automatically clicks "next," waits for the page to reload, and continues until all pages are processed. Includes safety limits and proper wait times to handle network latency.

Challenge: Two-Level Navigation

For large modules (100+ students), there's a top navigation that triggers full page reloads, which would kill the content script's state and execution context.

Solution: User-Guided Approach

Automatically handles bottom pagination (AJAX) but provides clear instructions for manual top navigation. The extension processes up to 100 students per run and includes group/page tracking in PDFs for easy organisation.

Challenge: Multiple Page Structures

Three different SITS views had completely different HTML structures, column orders, and field naming conventions, requiring flexible extraction logic.

Solution: Adaptive Detection

Implemented page type detection based on document.title with specialised extraction logic for each view. Handles field prefixes, column indices, and layout variations gracefully.

Challenge: Image Format Compatibility

Student photos from SITS come in various formats, but jsPDF has specific requirements. Some images were causing PDF generation to fail.

Solution: Canvas Conversion

All images are converted to base64 JPEG format using an HTML canvas before being embedded in the PDF, ensuring consistent format and compatibility.

Features

Smart Data Extraction

  • Automatically detects page type: "Students on Module", "Evision Module List photo retrieve", or "Student Search"
  • Extracts module code, module name, session, and CMIS group information
  • Handles field prefixes and cleans up data automatically
  • Gracefully handles missing photos and incomplete data

Automatic Pagination Handling

  • Resets DataTables to first page before extraction to ensure consistency
  • Automatically clicks through all available pages
  • Detects when pagination is complete by tracking student IDs
  • Includes safety limits (max 100 pages) to prevent infinite loops

Professional PDF Generation

  • Landscape A4 format with 20 cards per page (5 columns × 4 rows)
  • Intelligent headers with module info, group numbers, and page numbers
  • Automatic photo sizing and positioning with aspect ratio preservation
  • Handles long names and course titles with smart text wrapping
  • Descriptive filenames: ModuleCode_GroupName_page1.pdf

User Experience

  • Simple one-click operation from browser toolbar
  • Clear "About" section with usage instructions
  • Comprehensive console logging for debugging
  • Error handling with fallbacks for missing data

Lessons Learned

  • User Research is Essential: Understanding how university staff actually use SITS revealed edge cases (like two-level pagination) that weren't initially apparent
  • System Updates Require Flexibility: University systems change frequently; building adaptive selectors and detection logic is crucial for longevity
  • Timing Matters: AJAX pagination requires careful timing; too fast causes skipped pages, too slow frustrates users
  • Real-World Testing: Testing with actual university data revealed issues (duplicate injection, wrong column indices) that wouldn't appear in controlled environments
  • Documentation for Non-Technical Users: Clear usage instructions are as important as the code itself when your users are academics, not developers
