Business Improvement Projects Dashboard

Full-Stack Web Application Live in Production Work in Progress

A living catalogue of improvement work capturing the challenge, solution, and impact for business improvement projects across the University of Westminster. The system demonstrates end-to-end automation, combining SharePoint data management, Power Automate workflows, AWS S3 hosting, and modern web development to create a transparent, searchable project portfolio.

The Vision: Create a single, authoritative source for business improvement work that enables transparency, supports governance, and allows reuse of learning across teams. No more scattered documentation or lost knowledge when staff move on.

The Strategic Problem

Business improvement projects across the university faced several interconnected challenges:

  • Fragmented Documentation: Project information was scattered across emails, SharePoint sites, Teams channels, and Word documents, making it difficult to get an overview of ongoing work
  • Lack of Visibility: Senior leadership and stakeholders couldn't easily see what projects were in progress, their status, or their impact
  • Lost Knowledge: When staff members moved to different roles, project context and learning often disappeared with them
  • Inconsistent Quality: Without a standard approach, documentation quality varied significantly across projects
  • Manual Updates: Publishing project updates required manual copying, formatting, and uploading, leading to outdated information
  • Poor Governance: Difficult to demonstrate value, support decision-making, or report on portfolio health

The Solution: Automated Project Portfolio

I designed and developed an end-to-end system that addresses these challenges through automation and standardisation:

Architecture Overview

Data Source

SharePoint list serves as the single authoritative source. Project owners update information directly using familiar tools with validation and structure.

Automation Layer

Power Automate flow triggers on SharePoint changes, transforms data to JSON, and publishes to AWS S3 via pre-signed URLs—no manual steps required.

Web Interface

Static website hosted on S3 with real-time search, multiple filters (category, status, team), expandable project cards, and responsive design.

Rich Content

Supports markdown links and images, smart impact highlighting for measurable results, and color-coded categories and teams for quick visual scanning.

Key Technical Challenges

Challenge: Secure AWS Integration

Power Automate needed to upload files to S3 without exposing AWS credentials in the flow or storing them insecurely.

Solution: Pre-Signed URLs

Implemented a Python script that generates time-limited pre-signed S3 upload URLs. Power Automate calls this via HTTP, receives a secure URL, and uploads directly—credentials never exposed.

Challenge: Flow Stability

Power Automate flows would fail silently or trigger themselves recursively, creating infinite loops and unreliable publishing.

Solution: Defensive Automation

Used intermediate Compose steps to stabilise API responses, added explicit waits, prevented self-triggering with conditional logic, and implemented comprehensive error logging.

Challenge: Client-Side Performance

Loading and rendering 16+ projects with rich content, images, and multiple filters could slow down the interface, especially on mobile.

Solution: Optimised JavaScript

Implemented efficient filtering with data attributes, lazy loading for images, CSS animations instead of JavaScript for card transitions, and debounced search input.

Challenge: Rich Content Support

Project descriptions needed to include links and images without requiring staff to learn HTML or risk XSS vulnerabilities.

Solution: Markdown Parsing

Implemented secure markdown parsing for links [text](url) and images ![alt](url) with automatic conversion to HTML while escaping unsafe content. Added click-to-zoom lightbox for images.

Key Features

Multi-Dimensional Filtering

  • Filter by Theme: Projects can be categorized by functional area (e.g., technology, people, systems, student experience)
  • Filter by Status: Not Started, In Progress, Beta Testing, Complete
  • Filter by Originator: Track which teams or departments initiated each project
  • All filters work together - combine theme, status, and originator simultaneously

Real-Time Search

  • Search across all project fields: name, challenge, solution, impact, development notes
  • Updates results as you type with instant feedback
  • Shows result count (e.g., "4 of 16 projects")
  • Clear button to reset search instantly

Smart Impact Highlighting

  • Automatically highlights measurable impacts in red boxes (e.g., "16 projects documented in 1 location")
  • Detects time savings, percentages, and quantifiable metrics
  • Manual override with !! prefix for custom highlighting
  • Multi-line impacts displayed as clean bullet lists

Color-Coded Organisation

  • Each theme has a distinct color for quick visual identification
  • Originators displayed as colored badges for clear ownership
  • Status indicators with consistent color scheme across the interface
  • Accessibility-friendly color contrast throughout

Rich Content Support

  • Markdown links: [Link text](https://url.com) renders as clickable links
  • Markdown images: ![Description](image-url) embeds images inline
  • Click-to-zoom lightbox for all images with captions
  • Automatic escaping prevents XSS attacks while supporting rich formatting

Responsive & Accessible Design

  • Fully responsive layout works on desktop, tablet, and mobile
  • Semantic HTML structure for screen readers
  • Keyboard navigation support
  • Clean, professional University of Westminster branding

Development Journey

Phase 2: Automation Pipeline

  • Built Power Automate flow to export SharePoint data to JSON

Phase 3: Web Interface Development

  • Created static website with expandable project cards
  • Implemented filtering and search functionality
  • Added markdown parsing for links and images

Phase 4: Refinement & Features

  • Added team/department filtering after stakeholder feedback
  • Implemented color coding for project categories and teams
  • Created image lightbox for better content viewing
  • Fixed search security and performance issues

Lessons Learned

  • Start with Data Model: Agreeing on structure first made everything else easier. Simple, clear fields beat complex taxonomies.
  • Automation Requires Resilience: Power Automate flows need defensive programming—use intermediate steps, explicit waits, and prevent self-triggering.
  • Security by Design: Pre-signed URLs solved the credential exposure problem elegantly without complex infrastructure.
  • User Experience Matters: The best automation is invisible. Staff just update SharePoint; the system handles the rest.
  • Iterative Development Works: Started simple (static data) then added features (search, images, colors) based on usage and feedback.
  • Markdown is Powerful: Enabling links and images through simple syntax gave non-technical users rich formatting capabilities.
