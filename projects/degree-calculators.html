Degree Classification Calculators

React Web App WCAG 2.1 AA Accessible Live in Production

Interactive calculators that help students estimate their final degree classifications for undergraduate and postgraduate programmes. Students can explore "what if" scenarios to understand how future grades might affect their final classification. Built with accessibility at the core, featuring real-time calculations, light/dark mode, data persistence, and analytics tracking.

Try the UG Calculator Try the PG Calculator

The Student Need

Students at university frequently ask: "What degree classification am I on track to achieve?" While universities provide final results, there was no easy way for students to estimate their current standing based on completed modules, model "what-if" scenarios with predicted grades, understand how complex classification algorithms work, or access these calculations on any device including mobile.

Why it matters: Giving students visibility into their academic progress empowers them to make informed decisions about their studies, set realistic goals, and seek support when needed. Transparency in assessment criteria builds trust and reduces anxiety.

The Technical Challenge

Complex Classification Algorithms

Both undergraduate and postgraduate degree classifications use sophisticated algorithms that require careful implementation:

Undergraduate Calculator

Implements multi-stage calculations with credit weighting across different academic levels, minimum credit requirements, and logic to determine optimal scoring pathways based on completed modules.

Postgraduate Calculator

Handles credit-weighted averages across academic levels with different pass thresholds and classification boundary logic to determine final awards based on institutional assessment frameworks.

Accessibility Requirements

The calculators needed to meet WCAG 2.1 AA standards to ensure all students could use them regardless of disability, requiring screen reader compatibility with proper ARIA labels, sufficient colour contrast (minimum 4.5:1 for text, 3:1 for UI elements), keyboard navigation support, text reflow at 500% zoom without overlap, and semantic HTML with proper heading hierarchy.

The Solution

Core Features

The calculators provide dynamic module management with add/remove functionality and automatic recalculation, visual feedback through colour-coded sliders and results that change based on mark ranges, and data persistence using localStorage so students can save and track their progress over time. A light/dark mode toggle accommodates user preference and reduces eye strain, while real-time validation automatically checks credit requirements and mark ranges. The interface is fully responsive across phones, tablets, and desktops, with Google Analytics 4 integration to understand usage patterns and inform future improvements. All features meet WCAG 2.1 AA accessibility standards.

Technical Highlights

The project involved managing complex state across multiple modules with different credit levels and validation rules while ensuring real-time calculations and data persistence. The interface needed to be both visually engaging and fully accessible, which required combining colour-coded feedback with proper semantic HTML and ARIA attributes. To meet university deployment requirements, a custom build process was created that generates standalone HTML files with all CSS and JavaScript inlined for embedding in existing pages.

Analytics & Insights

Google Analytics 4 was integrated to track page views (understanding which calculator is used more frequently), save/load events (monitoring student engagement with data persistence), calculation sequences (debounced events to track usage patterns), and award type distribution (anonymised data on classification estimates). All analytics are fully anonymised—no personal data or module details are collected, only usage patterns to improve the tool.

Development Journey

Development proceeded through four iterative phases. Core functionality involved building React components with state management, implementing classification algorithms with test cases, creating a visual feedback system with colour-coded sliders, and adding a light/dark mode toggle. An accessibility audit led to fixing 9 identified WCAG issues, adding ARIA labels to sliders for screen reader support, improving colour contrast for text and icons, implementing proper heading hierarchy and semantic HTML, and enhancing link text with permanent underlines.

The analytics and persistence phase integrated Google Analytics 4 with event tracking, added save/load functionality with localStorage, implemented debounced sequence tracking, and added award type tracking for usage insights. Finally, deployment and export involved creating an automated build process for GitHub Pages, developing a standalone HTML file generator for embedding in university pages, configuring custom domain and HTTPS, and preparing the system for hosting on university servers.

Impact & Future Plans

The calculators are currently deployed on GitHub Pages and being prepared for hosting on university infrastructure for official embedding in course pages.

Planned Enhancements

Future enhancements include exporting results as PDF for student records, supporting multiple "what-if" scenarios with comparison view, integrating with the university's Student Information System for auto-populated data, enabling historical tracking of progress over terms, and conducting accessibility testing with assistive technology users to further refine the experience.

Lessons Learned

Building with WCAG standards from the start is far easier than retrofitting, though thorough auditing still revealed issues requiring iteration. Edge cases like extreme zoom levels and screen reader navigation only became apparent through comprehensive testing, reinforcing that user testing is essential. Automating the export of standalone files eliminated manual work and reduced deployment errors, while tracking usage patterns through analytics revealed how students actually use the tool versus initial assumptions. Clearly explaining the classification algorithms helps students understand and trust the results, demonstrating that transparency builds confidence.
