Degree Classification Calculators

React Web App WCAG 2.1 AA Accessible Live in Production

Interactive calculators that help students estimate their final degree classifications for undergraduate and postgraduate programmes. Built with accessibility at the core, featuring real-time calculations, dark mode, data persistence, and analytics tracking.

Try Undergraduate Calculator Try Postgraduate Calculator

The Student Need

Students at the University of Westminster frequently asked: "What degree classification am I on track to achieve?" While the university provides final results, there was no easy way for students to:

  • Estimate their current standing based on completed modules
  • Model "what-if" scenarios with predicted grades
  • Understand how the complex classification algorithms work
  • Access these calculations on any device, including mobile

Why it matters: Giving students visibility into their academic progress empowers them to make informed decisions about their studies, set realistic goals, and seek support when needed. Transparency in assessment criteria builds trust and reduces anxiety.

The Technical Challenge

Complex Classification Algorithms

Both undergraduate and postgraduate degree classifications use sophisticated algorithms that aren't immediately intuitive:

Undergraduate Algorithm

Requires minimum 120 credits at Levels 5 and 6. Calculates two provisional scores (excluding lowest Level 5 or Level 6 module), uses the higher score, with Level 5 weighted 1/3 and Level 6 weighted 2/3 of the best 220 credits.

Postgraduate Algorithm

Computes credit-weighted averages across Level 6 and Level 7 modules, requires different minimum marks for passing (40% vs 50%), and uses boundary thresholds for Pass, Merit, and Distinction classifications.

Accessibility Requirements

The calculators needed to meet WCAG 2.1 AA standards to ensure all students could use them, regardless of disability. This included:

  • Screen reader compatibility with proper ARIA labels
  • Sufficient colour contrast (minimum 4.5:1 for text, 3:1 for UI elements)
  • Keyboard navigation support
  • Text reflow at 500% zoom without overlap
  • Semantic HTML with proper heading hierarchy

The Solution

I developed two separate React-based calculators using Next.js, each tailored to its respective classification algorithm:

Core Features

  • Dynamic Module Management: Add/remove modules with automatic recalculation
  • Visual Feedback: Color-coded sliders and results that change based on mark ranges
  • Data Persistence: Save/load functionality using localStorage so students can track progress
  • Dark Mode: Theme toggle for user preference and reduced eye strain
  • Real-time Validation: Automatic checking for credit requirements and mark ranges
  • Mobile Responsive: Fully functional on phones, tablets, and desktops
  • Analytics Tracking: Google Analytics 4 integration to understand usage patterns

Accessibility Implementation

Achieved WCAG 2.1 AA compliance through:

  • Proper aria-label attributes on all interactive sliders
  • Improved delete icon contrast (slate-600/300 for 3:1+ ratio)
  • Enhanced result text contrast (-800/-300 colours for 4.5:1+ ratio)
  • H1 semantic heading for proper document structure
  • Descriptive link text ("Switch to Postgraduate Calculator" instead of "Click here")
  • Permanent underlines on links for colour-independent identification
  • Word-breaking in headings to prevent overlap at extreme zoom
  • Right padding to reserve space for theme toggle button

Key Technical Challenges

Challenge: Complex State Management

Managing multiple modules with different credit levels, marks, and validation rules while ensuring real-time calculations and data persistence.

Solution: React State + localStorage

Used React hooks for state management with automatic localStorage sync. Debounced analytics tracking to avoid excessive events while maintaining accurate usage data.

Challenge: Accessibility + Visual Appeal

Creating an interface that's both visually engaging with colour-coded feedback and fully accessible to screen reader users and those with colour blindness.

Solution: Multi-layered Feedback

Combined colour coding with text labels, ARIA attributes, and semantic HTML. Tested contrast ratios to meet WCAG standards while maintaining brand aesthetics.

Challenge: Static Export for Embedding

University web team needed standalone HTML files without external dependencies for embedding in existing pages.

Solution: Build Process Automation

Created custom Node.js script to inline all CSS and JavaScript into single-file HTML documents. Automated with npm run export command.

Challenge: Reflow at Extreme Zoom

At 500% zoom, long words like "Undergraduate" overlapped with the theme toggle button, failing WCAG 1.4.10 Reflow requirements.

Solution: CSS Word Breaking

Added break-words utility class and right padding to allow headings to wrap mid-word when necessary, preventing overlap at all zoom levels.

Analytics & Insights

Integrated Google Analytics 4 to track:

  • Page views: Understanding which calculator is used more frequently
  • Save/load events: Monitoring student engagement with data persistence
  • Sequence tracking: Debounced events to track calculation usage patterns
  • Award type distribution: Anonymised data on classification estimates

Privacy First: All analytics are anonymised. No personal data or module details are collected, only usage patterns to improve the tool.

Lessons Learned

  • Accessibility from Day One: Building with WCAG standards from the start is far easier than retrofitting, but thorough auditing still revealed issues that required iteration
  • User Testing is Essential: Edge cases like extreme zoom levels and screen reader navigation only became apparent through comprehensive testing
  • Build Processes Save Time: Automating the export of standalone files eliminated manual work and reduced deployment errors
  • Analytics Inform Design: Tracking usage patterns helps understand how students actually use the tool vs. how we assumed they would
  • Transparency Builds Trust: Clearly explaining the classification algorithms helps students understand and trust the results
