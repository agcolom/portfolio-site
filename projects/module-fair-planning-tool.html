Course Planner

jQuery Mobile Web App Used Since 2019 9 Degree Programmes

An interactive course planning tool that helps undergraduate students in Computer Science and Engineering make informed decisions about their module choices. Students can explore course structures, view detailed module information including assessments and prerequisites, and plan balanced semesters whilst tracking credit requirements in real-time. Many colleagues find this useful whilst discussing courses with their personal tutees, and course leaders find it useful to show the course structure at open days.

The Student Need

Choosing modules is one of the most significant decisions students make during their university journey, yet the process can be overwhelming. Students often face challenges understanding complex course structures across multiple academic levels, evaluating module options without clear information about content and assessments, planning balanced semesters whilst meeting credit requirements, and visualising how their choices fit into their overall degree pathway.

The Pedagogical Foundation

Student Agency and Informed Decision-Making: Research in higher education consistently shows that students who feel in control of their learning journey demonstrate higher engagement, persistence, and satisfaction. The Course Planner is grounded in self-determination theory, which emphasises autonomy, competence, and relatedness as key drivers of intrinsic motivation.

Transparency in Assessment: Making assessment patterns visible upfront reduces anxiety and helps students make strategic choices aligned with their learning preferences and strengths. This transparency builds trust and helps students take ownership of their academic planning.

Cognitive Load Management: By presenting complex information in an interactive, progressively disclosed format, the tool reduces cognitive overload. Students can explore at their own pace, adding and removing modules to see immediate feedback on their choices without needing to process all information at once.

Supporting Balanced Learning: The credit tracking and visual feedback system encourages students to create balanced semesters, promoting sustainable workload distribution—a key factor in student wellbeing and academic success.

Key Features

Interactive Course Structure Visualisation

Students select their degree programme from 9 available courses and instantly see their complete course structure organised by academic level (4, 5, 6) and semester. Core modules are clearly distinguished from optional modules, with pathway badges (e.g., "Mobile & Web Computing", "Data Science") helping students understand module specialisms.

Comprehensive Module Information

Each module card displays essential details including module code, name, credits, and semester. Swiping left reveals detailed assessment patterns (coursework, exams, practicals with weightings and qualifying marks), whilst swiping right shows quick-add buttons for optional modules. Clicking "more info" opens a full module description including learning outcomes, prerequisites, and module leader contact details.

Real-Time Planning and Validation

As students add or remove optional modules, the tool provides immediate visual feedback showing current credit totals for each level, required credits (120 per year), and colour-coded indicators (green tick for complete, amber warning for incomplete). Students can experiment with different combinations and see the impact on their overall structure instantly.

Email Integration

Once students have finalised their selections, they can email their module choices to themselves for future reference, making it easy to review offline, share with advisers, or use as a basis for formal module registration.

Technical Architecture

The tool uses a normalised data architecture that separates concerns and enables efficient maintenance. Module definitions, course structures, assessment patterns, and prerequisite rules are stored in separate JavaScript files, whilst a dedicated rendering engine handles all presentation logic. This modular approach means updating module information for a new academic year only requires editing data files, not touching the rendering code.

Data Structure

The system maintains four core data files. modules.js contains comprehensive module definitions (111 modules) including codes, names, descriptions, credits, levels, assessment patterns, and module leader information. courseStructures26.js defines course structures for 9 degree programmes, organised by level and semester with core/optional designations. assessments.js stores detailed assessment patterns with component types, weightings, and qualifying marks, whilst rules.js contains module prerequisites and corequisites.

User Interface

Built with jQuery Mobile for responsive, touch-friendly interactions, the interface features swipe gestures for revealing contextual actions, collapsible sections for semester organisation, and a colour-coded visual feedback system. The mobile-first design ensures the tool works seamlessly across devices from phones to desktop computers.

Maintenance and Evolution

Annual Updates

Each academic year, the tool is updated to reflect new modules, discontinued modules, and changes to course structures. The normalised data architecture means these updates are straightforward—editing the relevant data files without touching the presentation logic. This maintainability has been key to the tool's longevity.

Recent Enhancements (2025/26)

For the 2025/26 academic year, significant updates were made including adding a new BSc Creative Computing degree programme with 6 new modules, incorporating 7 additional missing modules across existing programmes, updating assessment patterns to reflect current university requirements, adding a prominent disclaimer notice about information currency, and fixing structural inconsistencies in course definitions.

Future Plans

University-Wide Expansion

The vision is to adapt this tool for all schools across the university, making informed module planning available to every undergraduate student. This would involve developing a generalised data schema that accommodates different course structures and assessment patterns, creating a centralised data management system for easy updates, integrating with the university's Student Information System (SITS) for real-time module availability, and building an administrative interface for non-technical staff to maintain module information.

Enhanced Features

Planned enhancements include prerequisite checking and validation to prevent invalid selections, timetable clash detection using live timetabling data, module popularity indicators to help students gauge demand, historical grade distributions to set realistic expectations, and integration with degree classification calculators to show how module choices might affect final awards.

Lessons Learned

Data architecture matters: Separating data from presentation logic has been crucial for maintainability. Updates that would have taken hours can now be completed in minutes.

Student feedback is invaluable: Regular informal feedback from students and academic advisers has driven continuous improvement. Features like swipe gestures and real-time credit tracking came directly from observing how students used early versions.

Mobile-first works: Many students access the tool during module fairs on their phones. Designing for touch interactions from the start made the desktop experience better too, not worse.

Transparency reduces anxiety: Students consistently report that seeing assessment patterns upfront helps them feel more confident about their choices and reduces pre-registration stress.

Pedagogical Impact: Beyond the technical achievement, this tool represents a commitment to student-centred design in higher education technology. By prioritising transparency, agency, and usability, it demonstrates how thoughtful EdTech solutions can meaningfully enhance the student experience.
